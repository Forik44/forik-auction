# 🎡 Аукцион аниме

Платформа для игры, где игроки ставят очки на аниме и крутят колесо аукциона.
Шанс победы пропорционален очкам, но розыгрыш идёт **на выбывание**: победитель
определяется заранее (строго по весам), а затем участники по одному вылетают с
прокрутом колеса — изначальные шансы при этом сохраняются точно.

Стек: **ASP.NET Core 8 + Blazor Server + SignalR + EF Core (SQLite)**. Один проект,
запускается на любом компьютере с .NET 8, легко переносится на хостинг.

---

## Возможности

- Вход через Google, вся информация сохраняется в базе.
- Бессрочные комнаты. Создатель задаёт пароль, друг заходит по коду + паролю.
- Вкладка **Аукцион**: игроки сами вписывают аниме и ставят очки (старт — 1000),
  видно, кто что поставил, проценты шанса и общее колесо. Колесо крутится
  **синхронно у всех** в комнате (через SignalR), время прокрутки круга
  настраивается, в конце красиво показывается победитель.
- Вкладка **Таланты**: дерево перков за «очки таланта» (ОТ), которые копятся
  каждый аукцион. Прокачка косвенно увеличивает очки.
- Вкладка **Квесты**: 4 (или больше с талантом) задания из реальной жизни на
  следующий аукцион; выполнил — получил очки. Всего в пуле 50 квестов.
- Наведение на игрока показывает подробную карточку: откуда взялись его очки,
  все бонусы и дебаффы, таланты и текущий баланс ОТ.

## Игровые механики

### Очки на аукцион
Стартовые очки игрока на каждый аукцион складываются из источников (всё это видно
в карточке игрока при наведении):

| Источник | Значение |
|---|---|
| Базовые очки | **1000** |
| Талант «Капитал» | +40 за уровень |
| Бонус проигравшему (прошлый аукцион) | **+200** (растёт с «Стипендией») |
| Штраф победителю (прошлый аукцион) | **−200** (уменьшается «Благородством») |
| Награды за выполненные квесты | сумма наград × множитель «Мотивации» |

Проигравший получает **+200** на следующий аукцион, победитель **−200** — ровно
как в исходной игре. Талант «Благородство» снижает штраф (например, «если соперник
выиграл — теряет меньше»), «Стипендия неудачника» увеличивает бонус отстающему.

### Очки таланта (ОТ) и дерево талантов
ОТ — отдельная валюта прокачки. Начисляются **каждый аукцион**: +3 всем участникам
и +2 дополнительно проигравшему (механика «резинки», чтобы отстающий догонял).

Стоимость уровней **растёт**, а у веток есть требования — поэтому невыгодно лить
всё в одну прокачку, лучше развиваться вширь и каждый аукцион решать, куда вложить.

Ветки:
- **Богатство** — «Капитал» (+40 базовых очков/ур.), «Инвестор» (неистраченные на
  ставку очки в конце аукциона конвертируются в ОТ — выбор: ставить или копить).
- **Сопротивление** — «Стипендия неудачника», «Благородство».
- **Квесты** — «Пытливый ум» (+1 квест/ур.), «Мотивация» (+15% к награде/ур.).
- **Удача** — «Фартовый» (+3%/ур. к твоему весу на колесе — шанс чуть выше, чем по
  чистым очкам).

Полный список — в `src/ForikAuction/Game/TalentCatalog.cs` (легко балансировать).

### Колесо на выбывание
1. Победитель выбирается сразу, пропорционально весам (очки × множитель «Фартового»).
2. Затем проигравшие выбывают по одному; чем меньше вес, тем выше шанс выбыть раньше
   (для драматизма). Поскольку победитель зафиксирован первым с верными
   вероятностями, **итоговое распределение остаётся честным**.

Это проверено тестом: при весах 700/200/100 победы выпадают ≈70/20/10%
(см. `tests/ForikAuction.Tests`).

### Квесты
50 несложных реальных заданий (спорт, доброта, продуктивность, общение, быт,
челленджи) в `src/ForikAuction/Game/QuestCatalog.cs`. На каждый аукцион выдаётся 4
случайных (или больше с «Пытливым умом»). Выполненные приносят очки (и иногда ОТ)
к следующему аукциону.

## Запуск на своём компьютере

### 1. Поставить .NET 8 SDK
Скачать с https://dotnet.microsoft.com/download/dotnet/8.0 и проверить:
```bash
dotnet --version   # 8.x
```

### 2. Настроить вход через Google
1. Зайти в **Google Cloud Console → APIs & Services → Credentials**.
2. Создать **OAuth client ID** типа *Web application*.
3. В **Authorized redirect URIs** добавить (порт совпадает с тем, на котором
   запускаете приложение):
   ```
   http://localhost:5080/signin-google
   https://localhost:7080/signin-google
   ```
4. Скопировать **Client ID** и **Client secret**.

Не храните секрет в `appsettings.json` для общего репозитория — используйте
user-secrets:
```bash
cd src/ForikAuction
dotnet user-secrets init
dotnet user-secrets set "Authentication:Google:ClientId" "ВАШ_CLIENT_ID"
dotnet user-secrets set "Authentication:Google:ClientSecret" "ВАШ_CLIENT_SECRET"
```
(для быстрой пробы можно просто вписать значения в `appsettings.json`).

### 3. Запустить
```bash
cd src/ForikAuction
dotnet restore
dotnet run
```
Откройте http://localhost:5080. База `forik.db` (SQLite) создастся автоматически.

## Чтобы друг тоже смог зайти

Самый простой способ — туннель: будет публичный HTTPS-адрес, который друг откроет
откуда угодно, а Google-вход продолжит работать.

**Вариант А — Cloudflare Tunnel (бесплатно, без аккаунта):**
```bash
cloudflared tunnel --url http://localhost:5080
```
Получите адрес вида `https://что-то.trycloudflare.com`. Добавьте
`https://что-то.trycloudflare.com/signin-google` в Authorized redirect URIs в
Google Console — и отправьте адрес другу.

**Вариант Б — Microsoft Dev Tunnels** (встроены в .NET/VS): `devtunnel host`.

**Вариант В — одна локальная сеть:** в `Properties/launchSettings.json` поменяйте
`applicationUrl` на `http://0.0.0.0:5080`, узнайте свой IP (`ipconfig`/`ifconfig`)
и дайте другу `http://ВАШ_IP:5080` (redirect URI в Google — с этим же адресом).
Туннель удобнее, т.к. даёт стабильный HTTPS.

## Перенос на хостинг

- **Сборка:** `dotnet publish -c Release`.
- **База:** для одного сервера SQLite достаточно (файл на постоянном диске). Для
  нагрузки переключитесь на PostgreSQL: пакет `Npgsql.EntityFrameworkCore.PostgreSQL`,
  в `Program.cs` `UseSqlite` → `UseNpgsql`, строка подключения — в переменную
  окружения `ConnectionStrings__Default`. Модель на EF Core, схема перенесётся.
- **Секреты:** `Authentication__Google__ClientId` / `__ClientSecret` через переменные
  окружения хостинга.
- **Redirect URI:** добавьте боевой домен `https://ваш-домен/signin-google`.
- За реверс-прокси (nginx и т.п.) включите forwarded headers и пропускайте WebSocket
  (нужно для SignalR).

## Проверка логики (тесты)

Игровое ядро (колесо, расчёт очков, таланты, квесты) вынесено в файлы без внешних
зависимостей и покрыто оффлайн-тестами:
```bash
cd tests/ForikAuction.Tests
dotnet run -c Release
```
Проверяется главное: распределение победителей колеса совпадает с весами, победитель
никогда не выбывает, корректность расчёта очков/бонусов/штрафов и каталога талантов
и 50 квестов.

## Структура проекта

```
ForikAuction.sln
src/ForikAuction/
  Program.cs                 — хост, Google-auth, маршруты, SignalR
  Data/                      — EF Core: AppDbContext + сущности
  Game/                      — чистая игровая логика (тестируемая):
      WheelEngine.cs           колесо на выбывание (сохраняет шансы)
      PointsCalculator.cs      стартовые очки + разбивка источников
      TalentEffects.cs         эффекты талантов
      TalentCatalog.cs         дерево талантов (баланс)
      QuestCatalog.cs          50 квестов
  Services/                  — RoomService, AuctionService, TalentService, хеш паролей
  Hubs/AuctionHub.cs         — реал-тайм комнаты и синхронная прокрутка
  Components/                — Blazor UI (Home, RoomPage, PlayerCard, layout)
  wwwroot/js/wheel.js        — отрисовка колеса (canvas) + SignalR-клиент
  wwwroot/css/app.css        — стили
tests/ForikAuction.Tests/    — оффлайн-проверка ядра
```
